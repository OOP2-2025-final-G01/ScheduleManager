## アプリ名
スケジュール管理アプリ

## 概要
本アプリは、カレンダーにTODOリストを書き込めるスケジュール管理アプリです。


【最終案】スケジュール管理
「個別のアプリ」ではなく、5人の機能がデータで繋がる1つのシステムを作ります。

TODOリストで立てた予定をタイマーと直結させ、作業時間を正確に記録できます。「昨日できなかったこと」をシステムが自動で検出し、翌日の画面に「リベンジ項目」として警告を出すことで、計画倒れを防ぎます。


本アプリは、チーム内で計画（TODO）→実行（タイマー）→追跡（履歴・リマインド）→分析（勉強ログ・グラフ）→統合（ダッシュボード）というデータのバトンタッチにより、日々の学習・作業の達成を支援します。

主な目玉機能:
- TODOで立てた予定をカレンダーに登録し、日にちごとにタスクを管理する
- TODO を選んで集中タイマーで作業時間を記録し、実績として保存する
- 昨日の未完了タスクを自動抽出して「やり残し」として翌日ダッシュボードに警告表示する
- 予定数と完了数を比較した達成率の可視化や、作業時間のグラフ表示による分析

視覚的な分析により、予定通りに進められたか、どれだけ集中できたかを客観的に確認できます。


## 各機能の説明

### TODOリスト
自分で新たなタスクとタスクごとの目標時間を追加することができます。

役割：冨田（TODOリスト担当）

機能概要：
- タスク作成、編集、削除（タイトル、予定日、所要時間など）
- カレンダーとの連携（指定日付にタスクを表示）
- タスクの完了フラグ（is_completed 0/1）を持ち、集計に使用する

実装メモ：現在 `init_db.py` で `todos` テーブル (task, due_date, duration, is_completed) を作成しています。
### 集中タイマー
タスクをクリックするとタイマーがスタートします。勉強時間はデータベースに保存され、目標時間を越えるとタスクを完了したことを示すメダルが付与されます。

役割：神納（集中タイマー担当）

機能概要：
- タスクを選択してタイマーを開始/停止するインターフェース
- 作業した時間をサーバーに記録し、タスクごとの実績時間を保存する
- タイマーは一時停止／再開が可能で、合計作業時間を算出する

連携ポイント：タイマー記録は後続の履歴／分析機能で使用されます。
### 履歴・リマインド


役割：神戸（履歴・リマインド担当）

機能概要：
- 日次でタスク履歴を集計し、未完了タスク（is_completed=0）を抽出する
- 昨日のやり残しをダッシュボードで警告表示する機能
- 必要に応じてリマインド通知（画面内のバッジやポップアップ）を表示する

### 勉強ログ・グラフ

役割：武藤（勉強ログ・グラフ担当）

機能概要：
- 期間ごとの達成率（予定数に対する完了数）を計算・可視化
- タイマーで記録された作業時間を時間軸のグラフ（折れ線/棒グラフ）で表示
- ダッシュボード上に「今月の達成率」などを表示するウィジェットを提供



## 使い方
app.py起動にあたって環境設定が必要な可能性があります。ターミナル上で以下のコードを実行してください

仮想環境を作って有効化
python3 -m venv .venv
source .venv/bin/activate

依存をインストール
pip install -r requirements.txt


起動後にブラウザで http://127.0.0.1:5001/ を開くとダッシュボード（`templates/index.html`）が表示されます。

備考：
- `init_db.py` を実行すると `main.db` が初期化され、`todos` テーブルが作成されます。
- `templates/index.html` に月間達成率を表示するドーナツチャートを追加済みです（`/api/stats/monthly` から集計データを取得します）。
